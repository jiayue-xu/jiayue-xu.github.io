添加rdma新设备
执行 ip addr 命令查看系统中所有的网络接口及其配置信息，包括：接口名称、状态、MAC 地址、IP 地址、IPv6 地址、广播地址、网络接口类型等。

执行rdma link add创建一个名为 siw0 和 siw0_lo 的 RDMA 网络接口，具体解释如下：

  • rdma：表示 RDMA 子系统，用于管理 RDMA 网络接口和 RDMA 设备。
  • link：表示创建一个新的 RDMA 网络接口。
  • add：表示添加一个新的 RDMA 网络接口。
  • siw0：表示要创建的 RDMA 网络接口的名称。
  • type siw：表示要使用的 RDMA 驱动程序类型为 siw，即 Softiwarp 驱动程序，它提供了一种基于软件的 RDMA 解决方案。
  • netdev eth0：表示底层使用的网络设备为 eth0，即物理网卡的名称。

测试环境

rping

rping 是一个基于 RDMA（Remote Direct Memory Access）的网络测试工具，用于测试 RDMA 网络接口的性能和可靠性。它可以在两台或多台支持 RDMA 的计算机之间进行测试，以评估 RDMA 网络接口的吞吐量、延迟、带宽等指标。

rping 工具是由 Mellanox 公司开发的，它基于 InfiniBand 和 RoCE 技术，支持 RDMA 的数据传输方式。它可以在 Linux 和 Windows 系统中使用，支持多种传输模式和参数配置，如双向传输、单向传输、无超时、有超时等。

使用 rping 工具进行 RDMA 网络接口测试，需要在测试前先安装和配置相应的 RDMA 驱动程序和软件包，以确保系统支持 RDMA 功能。然后，可以在两台或多台计算机之间执行 rping 命令，进行 RDMA 测试，以评估网络接口的性能和可靠性。

  • -s：表示启动服务器端，用于监听客户端的连接请求。
  • -a 127.0.0.1：表示要监听的 IP 地址，这里指定为本地回环地址 127.0.0.1，也就是只监听本地连接请求。

SoftiWARP

为了测试一些rdma example测试程序，在Ubuntu 22上配置了SoftiWARP。
SoftiWARP是一种开源的InfiniBand软件，它提供了一种基于UDP / IP网络的虚拟InfiniBand网络。它允许在没有物理InfiniBand网络的情况下在计算机之间传输高速数据，而且它具有低延迟和高带宽的特点。SoftiWARP是Linux内核的一部分，并且已经被广泛使用在高性能计算领域、云计算、虚拟化以及大规模数据中心等场景中。SoftiWARP还支持许多InfiniBand协议，如Verbs、IPoIB、SRP等，可以实现高效的数据传输。
SoftiWARP是一个软件RDMA设备，对于任何RDMA开始编程,RDMA-enabled硬件可能不是。SoftiWARP是一个非常有用的工具，可以设置RDMA环境，并进行编码和实验。它也是教育和教授RDMA编程给学生的非常有用的工具。
SoftiWARP (siw)是用于Linux的软件iWARP内核驱动程序和用户库。它完全在软件上实现iWARP协议套件(MPA/DDP/RDMAP, IETF-RFC 5044/5041/5040)，不需要任何专用的RDMA硬件。SoftiWarp的目标是与OpenFabrics (OFA)生态系统集成。对于OFA集成，它是根据其内核和用户级接口编写的。SoftiWARP支持用户级和内核级应用程序。它使用OFA连接管理器来建立连接。内核组件运行在TCP内核套接字之上。
