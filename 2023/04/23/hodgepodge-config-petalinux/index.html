发表于 2023-04-23 更新于 2023-05-11 分类于 hodgepodge

Xilinx MPSoC启动流程

  1. 加载FSBL(First Stage Boot Loader) - 当设备上电或者复位时，硬件会自动从flash中加载FSBL，FSBL是一个预制的引导程序，它的作用是初始化硬件、设置时钟、检测系统状态等，并且加载bitstream文件到FPGA芯片中。
  2. 加载ATF(ARM Trusted Firmware) - FSBL加载bitstream后，会将控制权转移给ATF，ATF是由ARM维护的可信固件，它的作用是初始化CPU、设置内存、加载U-Boot等。
  3. 加载U-Boot - ATF加载完成后，会将控制权转移给U-Boot,U-Boot是一种开源启动程序，它的作用是初始化硬件、加载Linux内核和设备树等信息，并设置启动参数，最终将控制权转交给Linux内核。
  4. 加载Linux内核 - U-Boot将控制权转交给Linux内核，Linux内核是操作系统的核心，它的作用是完成系统初始化、加载驱动程序和应用程序等，最终启动Linux操作系统。

Xilinx提供了Bootgen工具，Bootgen可以将上述FSBL、ATF、U-Boot、Linux内核、设备树等文件打包生成启动镜像，最终将其写入flash等存储设备中。

PetaLinux

PetaLinux是Xilinx公司推出的一款基于Yocto Project 的嵌入式 Linux 开发工具，旨在简化嵌入式 Linux 系统的开发和定制。PetaLinux 提供了一套完整的工具链，包括交叉编译工具链、构建系统、软件包管理器、设备树编辑器等，可以方便地进行 Linux 内核和文件系统的构建、配置、调试和部署。
PetaLinux 支持多种硬件平台，其中包括 Xilinx 的 Zynq、Zynq UltraScale+ MPSoC 等 SoC 平台，以及 Virtex、Kintex、Artix 等 FPGA 平台。PetaLinux 可以与 Xilinx Vivado 集成，实现 FPGA 和 Linux 内核的联合设计和调试。
PetaLinux 的主要功能包括：

  1. 嵌入式 Linux 系统的构建和定制：PetaLinux 提供了一整套工具链，可以方便地构建和定制嵌入式 Linux 系统，包括 Linux 内核、文件系统、设备驱动等。

  2. 设备树编辑器：PetaLinux 集成了设备树编辑器，可以方便地编辑设备树文件，实现硬件和软件的匹配和配置。

  3. 软件包管理器：PetaLinux 提供了软件包管理器，可以方便地添加、移除和更新软件包，支持多种软件包格式和源。

  4. 交叉编译工具链：PetaLinux 提供了一套交叉编译工具链，支持多种处理器架构和操作系统，可以方便地进行开发和调试。

  5. Vivado 集成：PetaLinux 可以与 Xilinx Vivado 集成，实现 FPGA 和 Linux 内核的联合设计和调试，提高开发效率。

Petalinux打包镜像

petalinux-build命令用于编译PetaLinux项目，会生成多个文件和目录：

  • images/linux目录 - 包含了生成的Linux内核、设备树和启动镜像等文件。
  • components/plnx_workspace目录：包含了编译过程中的中间文件和工作区。
  • build目录：包含了编译过程中生成的编译日志和其他文件。
  • rootfs目录：包含了根文件系统的文件和目录，用于构建 Linux 系统的根文件系统。
  • project-spec目录：包含了 PetaLinux 项目的配置文件和自定义组件等信息。

petalinux-package --boot命令用于将PetaLinux项目打包成启动镜像，可以烧写到闪存或SD卡中启动设备。该命令需要指定以下参数：

  • –fsbl - 指定FSBL文件的路径，FSBL是Zynq MPSoC设备启动的第一个阶段，用于初始化硬件和加载bitstream。
  • –fpga - 指定bitstream文件的路径，如果使用了FPGA，则需要将bitstream文件加载到FPGA中。
  • –u-boot - 指定U-Boot文件的路径，U-Boot是设备启动的第二个阶段，在FSBL启动完成后被加载并执行。
  • –force - 强制覆盖已存在的镜像文件。

Xilinx SDK打包镜像

  • zynqmp_fsbl.elf: 是Xilinx Zynq MPSoC平台上的(FSBL)文件，它是启动Zynq MPSoC设备时第一个被执行的程序。FSBL主要工作是初始化硬件系统，加载bitstream(如果使用可重构逻辑FPGA的话)，执行一些基本的安全性检查，最后跳转到第二阶段的u-boot或者操作系统的启动程序。
  • pmu.elf: 用于在ARM架构的处理器上启用性能计数器和性能分析功能。PMU代表性能监控单元，它是ARM架构中的一个硬件模块，用于收集和分析处理器的性能数据。pmu.elf文件包含了一组用于配置和控制PMU的指令，以及一些用于读取和分析性能计数器数据的工具。在处理器上运行pmu.elf文件可以启用性能计数器和性能分析功能，并收集处理器的各种性能数据，例如指令执行次数、缓存命中率、分支预测成功率等等。

在PetaLinux开发流程中，zynqmp_fsbl.elf和pmu.elf是通过Xilinx SDK工具链编译生成的，在PetaLinux项目中通常位于<project>/images/linux目录下。

  1. 打开Xilinx SDK，新建一个workspace，选择File->New->Application Project，在弹出的对话框中的Hardware Platform中选择New，选择HDF创建新的硬件平台。

  2. 创建FSBL：配置好之后，选择Next->ZynqMP FSBL，Finish，通过fsbl_bsp->system.mss更改BSP属性。

  3. 创建PMU：使用第一步创建的硬件平台，通过fsbl_bsp->system.mss更改BSP属性。

  4. 打包BOOT.bin：由fsbl.elf、pmu.elf、bit、bl.elf、u-boot.elf、image.ub组成，选择Xilinx->Create Boot Image导入。

  5. 烧入FLASH：选择Xilinx->Program Flash，正常烧入需要30分钟。
