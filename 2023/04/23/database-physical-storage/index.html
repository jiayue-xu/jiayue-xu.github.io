数据库的物理存储模型

发表于 2023-04-23 更新于 2023-04-24 分类于 数据库 ， hodgepodge

物理存储介质

在大多数计算机系统中存在几种类型的存储介质，这些存储介质根据存取数据的速度、购买介质单位数据的成本以及介质的可靠性进行分类:

  • Cache - 高速缓存是最快和最昂贵的存储形式，缓存相对较小，由计算机系统硬件管理。
  • Main Memory - 主存储器存放可操作数据和机器指令。个人电脑的主存大概为几百GB大小，大型服务器系统主存可以达到数千GB大小。对于需要存储大量数据的数据库来说，主存通常太小(或太昂贵)，但随着主存的增大，许多企业数据库也可以放在主内存中。然而，主存作为易失性存储器，在电源故障或系统崩溃的情况下内容会丢失。
  • Flash Memory - 闪存作为非易失性存储器，即使电源关闭或发生故障，存储的数据也会被保留。闪存的每字节成本比主存低，但比磁盘的每字节成本高。闪存被广泛用于照相机和手机等设备的数据存储，闪存也用于在“USB闪存驱动器”中存储数据，即我们常用的“U盘”。随着价格的降低，在个人计算机中，用闪存来替代磁盘已经是很常见的事情。固态硬盘(solid-state disk, SSD)内部使用闪存存储数据的同时，提供类似于磁盘的接口，允许以块为单位存储或检索数据，这样的接口称为面向块的接口，块大小通常从512B到8KB节不等。
  • Magnetic-disk storage - 长期在线存储数据的主要介质是磁盘驱动器，也称为硬盘驱动器(hard-disk driver, HDD)。磁盘和闪存一样，是非易失性的。要访问存储在磁盘上的数据，系统首先将数据从磁盘移动到主存储器，从主存储器可以访问这些数据。当系统执行指定的操作后，需要将修改过的数据写入磁盘。尽管比ssd便宜得多，但就每秒支持的数据访问操作数量而言，磁盘的性能较低。

存储器接口

高速接口

磁盘和固态硬盘可以通过高速互连接口与计算机系统中的其他部件连接：

  • SATA(Serial ATA)接口 - SATA-3版本理论上支持6GB/s，允许数据传输速度高达600MB/s。
  • SAS(Serial Attached SCSI)接口 - SAS接口在服务器领域应用的更加广泛，SAS-3支持12GB/s的数据传输速率。
  • NVMe(Non-Volatile Memory Express) - 为了更好地支持SSD，2010年左右提出的NVMe接口标准，通常与PCIe接口一起使用。

SAN

当磁盘和固态硬盘与计算机系统中的其他部件的物理距离很远时，需要通过高速网络将磁盘连接到计算机系统。因此，提出了存储区域网络(storage area network, SAN)架构，大量的磁盘和数个服务器通过高速网络互连，这些磁盘通过独立磁盘的冗余阵列(redundant arrays of independent disks, RAID)技术为服务器提供逻辑上大而可靠的磁盘存储。SAN中，常见的网络互连技术：

  • iSCSI - 通过IP发送SCSI命令
  • FC(Fiber Channel) - 支持1.6~12GB/s的传输速率
  • InfiniBand - 低延时、高带宽网络

NAS

NAS(Network attached storage)是SAN的替代方案，它利用网络文件系统协议(NFS)或CIFS提供了文件系统接口。

闪存

与磁盘相比，SSD提供了更高的性能，但是
有两种类型的闪存：

  • NOR flash - NOR Flash和NAND Flash都是非易失性存储器，它们都可以用于嵌入式系统和移动设备等应用中。它们的工作原理和应用场景有所不同。

NOR Flash是一种串行存储器，它的存储单元可以直接寻址。它的读取速度比较快，因为数据可以直接从存储单元读取。NOR Flash还支持随机访问，因此适用于嵌入式系统中的代码存储和执行，例如固件存储和引导加载器等。NOR Flash的缺点是擦除速度比较慢，而且价格相对较高。

NAND Flash是一种并行存储器，它的存储单元不能直接寻址。它的读取速度比较慢，因为需要将数据从存储单元读取到缓冲区，然后再传输到CPU。但是，NAND Flash的擦除速度比较快，且价格相对较低。因此，NAND Flash适用于大容量数据存储，例如移动设备中的媒体文件和数据文件等。

总体来说，NOR Flash适合于代码存储和执行，而NAND Flash适合于数据存储和读取。在实际应用中，NOR Flash和NAND Flash也可以组合使用，以满足不同的存储需求。
